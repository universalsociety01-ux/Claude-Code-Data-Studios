# /prototype

## Purpose
Quick proof-of-concept to validate an approach.

## Workflow
1. Define success criteria
2. Identify minimal data needs
3. Implement simplest viable version
4. Validate against criteria
5. Document findings and limitations
6. Recommend next steps

## Agents Involved
- research-lead (approach)
- Relevant specialist (implementation)

## Inputs
- Concept description
- Available data

## Outputs
- Prototype notebook in notebooks/experiments/
- Findings with go/no-go recommendation
