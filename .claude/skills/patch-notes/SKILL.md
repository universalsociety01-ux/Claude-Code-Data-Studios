# /patch-notes
## Purpose
Write user-facing patch notes for model or pipeline updates.
## Workflow
1. Gather changes since last release
2. Highlight model performance changes
3. Document breaking changes
4. Write clear, non-technical summary
## Agents Involved
- documentation-specialist (writing), producer (review)
## Inputs
- Version
## Outputs
- Patch notes document
## Example Usage
```
/patch-notes --version v2.1.0
```
