# /tech-debt

## Purpose
Identify, categorize, and prioritize technical debt.

## Workflow
1. Scan for TODOs, complexity, outdated deps, missing tests
2. Categorize by severity
3. Estimate remediation effort
4. Prioritize by impact
5. Generate tech debt tracker

## Agents Involved
- producer, testing-specialist, performance-engineer, security-engineer

## Inputs
- Optional: specific directory to audit

## Outputs
- Tech debt tracker from template
- Prioritized remediation plan
