# /design-review

## Purpose
Review ML system designs, pipeline architectures, data models.

## Workflow
1. Read design document
2. Check completeness against template
3. Evaluate architecture decisions
4. Identify risks and gaps
5. Assess operational feasibility

## Agents Involved
- data-director or ml-director, relevant lead, mlops-lead

## Inputs
- Design document path

## Outputs
- Completeness checklist
- Risk assessment and architecture feedback
