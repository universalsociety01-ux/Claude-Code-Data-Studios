# /map-systems
## Purpose
Create a comprehensive map of all data systems, their dependencies, and data flows.
## Workflow
1. Scan codebase for data sources and sinks
2. Trace data flow through pipelines
3. Identify dependencies between systems
4. Map upstream and downstream consumers
5. Generate system map documentation
## Agents Involved
- data-director (oversight), data-engineering-lead (technical), documentation-specialist (documentation)
## Inputs
- None (auto-scans project)
## Outputs
- System dependency map, data flow diagram
## Example Usage
```
/map-systems
```
