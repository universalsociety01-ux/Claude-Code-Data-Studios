# /bug-report
## Purpose
Create a structured bug report for data quality, pipeline, or model issues.
## Workflow
1. Gather reproduction steps
2. Identify affected systems
3. Assess severity and impact
4. Document expected vs actual behavior
5. Generate structured bug report
## Agents Involved
- producer (coordination), relevant specialist (investigation)
## Inputs
- Bug description
## Outputs
- Structured bug report
## Example Usage
```
/bug-report "Model predictions suddenly all returning 0.5"
```
