# /scope-check

## Purpose
Evaluate scope, complexity, and feasibility before committing resources.

## Workflow
1. Parse proposal description
2. Break into subtasks with dependencies
3. Identify required data, compute, expertise
4. Estimate effort (optimistic/likely/pessimistic)
5. Flag risks
6. Provide go/no-go recommendation

## Agents Involved
- producer (methodology)
- Relevant lead (feasibility)
- cost-analyst (resources)

## Inputs
- Task description
- Optional: deadline, resources

## Outputs
- Scope assessment with subtask breakdown
- Risk register and recommendation
