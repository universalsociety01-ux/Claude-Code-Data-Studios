# /reverse-document
## Purpose
Analyze existing code and generate comprehensive documentation.
## Workflow
1. Scan codebase structure
2. Analyze code purpose and dependencies
3. Generate module documentation
4. Create architecture overview
5. Write data flow diagrams
## Agents Involved
- documentation-specialist (writing), relevant leads (review)
## Inputs
- Directory or file path
## Outputs
- Generated documentation in docs/
## Example Usage
```
/reverse-document src/pipelines/
```
