# Review Workflow

## Four Perspectives
1. Standards Compliance — PEP 8, types, docs, naming
2. Architecture — separation of concerns, patterns, modularity
3. Domain-Specific — mapped to path-scoped rules
4. Security — secrets, PII, injection, validation

## Verdicts
- APPROVE — ready to merge/deploy
- REQUEST CHANGES — blocking issues
- DISCUSS — needs architectural discussion

## Data Review
Schema validation, quality checks, lineage, PII scan.

## Model Review
Evaluation metrics, fairness audit, model card, reproducibility, no leakage, deployment readiness.
